# 1. Problem & Scope

## 1.1 Problem Statement

目前 Automation 模板只能在系統內建立，無法跨工作區搬移。當使用者想把已建立好的模板分享給其他工作區、或從 AI 生成的設計轉成可用模板時，都缺乏標準格式可走。這導致使用者只能手動重建模板內容，且未來模板市集、跨組織分享等功能都缺乏共同的資料載體。

## 1.2 Background & Context

既有 Automation PRD 已定義模板的執行邏輯（節點、連線、條件），但對「模板如何在系統間流動」沒有規範。

隨著 AI 生成 Automation、未來模板市集等需求出現，需要先定義一份共通的「可攜帶資料結構」作為基礎。本文件**先於資料庫設計**，描述領域模型（模板「概念上由什麼組成」），不涉及 table、外鍵、索引等實作細節。

本 POC 是後續多個 feature 的基礎：未來的「持續性同步匯入」、「跨工作區模板引用」、「模板市集」都會基於本 schema 擴充。

## 1.3 Target Users / Personas

- **PM / 流程設計者**：建立 Automation 模板的人。目標是設計可重複使用的流程；痛點是無法把已建好的模板搬到其他工作區或分享給同事
- **AI 生成功能使用者**：透過 AI 自動產出模板的使用者。目標是 AI 產出能直接套用到編輯器；痛點是現有 AI 結果格式不統一，無標準寫入路徑
- **系統管理員 / RD**：管理系統內建模板的人。目標是提供官方範本給新工作區；痛點是沒有標準格式可批次載入

## 1.4 Success Criteria

**判定標準**：使用者可以將模板匯出為 JSON、匯入到其他工作區成功使用；AI 生成功能可以透過同一條路徑寫入資料庫；系統內建模板可由 RD 透過 seed 機制載入。

**量化指標**：
- POC 階段：匯入成功率 > 95%（在 schema 合規的檔案下）
- POC 階段：匯入後模板可完整在編輯器顯示，與原模板結構一致
- 上線 3 個月：> 30% 的新建模板來自匯入路徑（含 AI 生成 + 檔案匯入）

**反指標**：
- 匯入流程不應增加既有「自行建立」流程的步驟或複雜度
- 匯入失敗時不留下半成品資料（atomicity）

## 1.5 Scope

**In scope**：
- Automation 模板的領域模型定義（JSON Schema）
- 匯出模板為檔案（.json）
- 從檔案匯入模板
- AI 生成結果寫入資料庫（成為一筆模板）
- 系統內建模板由 RD 放入 seed 資料夾，啟動時載入

**Out of scope**：
- 從外部資料庫匯入 task — 延後處理，避免 POC scope 膨脹
- 持續性同步 / 增量匯入 — 屬於進階整合，需獨立設計
- 跨工作區的模板引用 — POC 不支援，匯入即複製
- 模板版本歷史的匯出 — POC 只匯出當前最新版
- 模板執行引擎 — 屬於既有 Automation PRD 範圍
- 後台模板管理介面 — POC 階段 RD 手動操作 seed 資料夾
- 匯出檔案加密 / 簽章 — POC 範圍不涉及交易場景

**Future（未來方向）**：
- 持續性匯入綁定資料來源
- 模板市集（使用者可發布模板供他人下載）
- Schema 升版機制（舊版檔案自動 migrate）
- 跨組織模板引用（含驗證來源組織身份）

### 1.5.1 POC 級設計決定

| 議題 | POC 決定 | 未來可能調整 |
|---|---|---|
| 攜帶格式 | JSON | 增加 YAML、二進位格式 |
| 匯入來源 | 檔案上傳（.json） | 增加貼上 JSON、分享連結匯入 |
| 執行人處理 | Placeholder（角色 / 變數），不帶具體 user_id | 增加「匯入時自動 mapping」精靈 |
| Task 結構引用 | 內嵌完整定義（self-contained） | 改為外部引用 + 版本鎖 |
| 模板版本 | 只匯出最新版，不帶歷史 | 增加多版本匯出 |
| 驗證 | 匯入時做 schema 驗證 | 增加語意驗證（連線是否合理等） |
| 加密 / 簽章 | 不做（純 JSON） | 涉及交易或跨組織分享時再評估 |
| 系統內建模板 | RD 放 seed 資料夾，啟動時載入 | 後台管理介面、版本管理、熱度排序 |

> 上表將於 §7 展開成正式 ADR。

## 1.6 Assumptions & Constraints

- 使用者已在系統中認證且有所屬工作區
- 既有 Automation 模板結構（節點 / 連線 / 條件）已定義
- 模板執行引擎不在本 feature 範圍，本 feature 只處理「模板資料的攜帶」
- POC 階段檔案大小上限 5 MB
- JSON 為標準格式，未來可能擴充 YAML 但不在 POC
- 匯入後使用者需在編輯器中手動指派執行人 placeholder 對應的具體成員
- 既有 Automation 模板的「版本」、「啟用 / 草稿」概念延用，本 feature 不重新定義
